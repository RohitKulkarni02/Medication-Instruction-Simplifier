# Medication Instruction Simplifier — Project documentation

CS6180 · Group 14. This document describes everything in the repository as of the latest update: purpose, layout, the extraction pipeline, data expectations, tests, and how to run things locally.

---

## Purpose

The project aims to support **medication instruction simplification** (e.g., making FDA label text easier for patients to understand). The code checked into this repo so far is **Layer 1: structured extraction** — it does **not** call large language models. It normalizes label text from two different JSON shapes into **one unified schema** so original and simplified labels can be compared or evaluated on the same fields.

For the broader research goals and methodology, see **`CS6180_Project_Proposal.pdf`** in the repository root.

## `extract_labels.py` — behavior

### Role

- Reads a **JSON array** of drug records from disk.
- Maps each record into a **unified output schema** with five safety-related text fields plus metadata.
- Writes a single JSON output file and prints a **summary** (and per-record **warnings** when fields are missing or empty).

### Unified output fields (per drug)

| Output key | Meaning |
|------------|---------|
| `drug_name` | From input `drug_name`, or `"UNKNOWN"` if absent. |
| `source` | `"openfda"` or `"simplified"` depending on pipeline. |
| `extraction_method` | Always `"structured"`. |
| `extraction_confidence` | Always `"high"`. |
| `boxed_warning` | Boxed warning text. |
| `dosage` | Dosage / administration text. |
| `warnings` | Warnings section. |
| `contraindications` | Contraindications. |
| `interactions` | Drug interactions. |

Missing keys, empty strings, or whitespace-only strings for any of the five safety fields become JSON **`null`** in the output. A line is printed for each such case: `WARNING: <drug_name> missing field: <field>`.

### Input key mapping

**Original pipeline** (`--source original`, input `drug_labels.json`):

| Output | Input key in `drug_labels.json` |
|--------|----------------------------------|
| `boxed_warning` | `boxed_warning` |
| `dosage` | `dosage_and_administration` |
| `warnings` | `warnings` |
| `contraindications` | `contraindications` |
| `interactions` | `drug_interactions` |

**Simplified pipeline** (`--source simplified`, input `simplified_labels.json`):

| Output | Input key |
|--------|-----------|
| (same five fields) | `boxed_warning`, `dosage`, `warnings`, `contraindications`, `interactions` |

### CLI usage

```bash
python extract_labels.py --source original
python extract_labels.py --source simplified
```

| `--source` | Reads | Writes |
|------------|--------|--------|
| `original` | `drug_labels.json` | `extracted_original.json` |
| `simplified` | `simplified_labels.json` | `extracted_simplified.json` |

### Public functions (for tests and reuse)

- `extract_record(record, field_map, source)` — map one dict through a field map and attach metadata.
- `extract_original(data)` — list comprehension over `ORIGINAL_FIELD_MAP`, `source="openfda"`.
- `extract_simplified(data)` — same for `SIMPLIFIED_FIELD_MAP`, `source="simplified"`.
- `print_summary(results)` — prints totals and null-field statistics for a list of extracted records.

Constants `ORIGINAL_FIELD_MAP`, `SIMPLIFIED_FIELD_MAP`, and `SAFETY_FIELDS` define the schema and which keys participate in the summary.

## Tests (`test_extract_labels.py`)

Tests use **pytest**. They cover:

- **`extract_record`**: full mapping for original and simplified samples; metadata; missing / empty / whitespace fields → `None`; missing `drug_name` → `"UNKNOWN"`; simplified uses `interactions` not `drug_interactions`.
- **`extract_original` / `extract_simplified`**: output length matches input; every row has the correct `source`.
- **`print_summary`**: mocked `print`; checks for “all fields present”, per-field null counts, and combined totals when one of two records has nulls.

They do **not** cover: file I/O, `main()` / argparse, or the warning lines emitted by `extract_record`.

### Running tests

```bash
pip install -r requirements.txt
pytest test_extract_labels.py -v
```
